

# css学习笔记

## css简介
CSS 指层叠样式表 (**C**ascading **S**tyle **S**heets)，样式定义**如何显示** HTML 元素。样式通常存储在**样式表**中，把样式添加到 HTML 4.0 中，是为了**解决内容与表现分离的问题**。**外部样式表**可以极大提高工作效率，外部样式表通常存储在 **CSS 文件**中，多个样式定义可**层叠**为一个。文件后缀为.css

## *CSS语法*

### css示例

CSS 规则由两个主要的部分构成：选择器，以及一条或多条声明:
![img](https://www.runoob.com/wp-content/uploads/2013/07/632877C9-2462-41D6-BD0E-F7317E4C42AC.jpg)

- 选择器通常是您需要改变样式的 HTML 元素。
- 每条声明由一个属性和一个值组成。
- 属性（property）是您希望设置的样式属性（style attribute）。每个属性有一个值。属性和值被冒号分开。
- CSS声明总是以分号 **;** 结束，声明总以大括号 **{}** 括起来:
p {color:red;text-align:center;}
- 为了让CSS可读性更强，你可以每行只描述一个属性:
p
{
    color:red;
    text-align:center;
}

### 样式表

#### 内联样式

在相关的标签内使用<style>属性。（style属性可以包含任何css属性）

<p style="background:red;font-size:40px;">CSS<p>

#### 内部样式

在<head>标签内定义内部样式表可直接在<body>里应用

p
{

  color:red;
  text-align:center;

}

#### 外部样式

样式需要应用在多个页面时，可以通过改变一个文件来改变整个站点的外观。单独创建一个.css的文件对标签进行声明，每个页面使用<link>标签链接到样式表。<link>标签在<head>中。

<link rel="stylesheet" type="text/css" href"xxx.css">

### 选择器

#### 选择器优先级

行内样式>ID选择器>类选择器>元素选择器

#### 全局选择器

可以与任何元素匹配，优先级最低，一般做样式初始化

*{

  color:red;
  text-align:center;

}

p
{

  color:red;
  text-align:center;

}

下面的对<p>单独生效，因为下面的优先级大于上面的优先级。

#### 元素选择器

HTML文档中的元素p、b、div、ing、body等。

标签选择器，选择的是页面上所有这种类型的标签，所以经常描述“共性”，无法描述某一个标签的“个性”

p{

​	font-size：10px；

}

如果想让标签中的某部分修改样式，可用<span>标签把修改部分围起来，然后给<span>标签加一个标签选择器

<p>学完了<span>前端</span>,继续学Java</p>
span{
	color:red;
    }

#### 类选择器

规定用圆点来定义，针对想要用的所有标签使用

<style>
    .classone{
        color:red;
    }
</style>
<p class="classone">
    hello
</p>

##### class属性特点

1. 类选择器可以被多种标签使用

2. 类名不能以数字开头

3. 同一个标签可以使用多个类选择器，用空格隔开

   <P class="classone classtow">hello</p>

#### ID选择器

针对某一个特定标签来使用，只能用一次，.css中的ID选择器以#来定义

<p id="mytitle">hello</p>

#mytitle{

​	border:3px;

}

*注*：

1. id是唯一的（与class区分）
2. id不能以数字开头

#### 合并选择器

语法：选择器1，选择器2{}

作用：提取共同样式，减少重复代码

p，h{

​	color:red;

}

## 关系选择器

### 后代选择器

（Descendant Selector）用于选择**作为某元素后代**的所有子元素，通过空格  连接父级与子级选择器，层级可嵌套。

| 项目          | 内容                                                         |
| ------------- | ------------------------------------------------------------ |
| **语法**      | `E F { ... }` （E 是父级元素，F 是后代元素）                 |
| **HTML 结构** | `html <ul> <li>宝马</li> <li>奔驰</li> </ul> <ol> <li>奥迪</li> </ol> ` |
| **CSS 代码**  | `css ul li { color: green; } `                               |
| **效果说明**  | 只选中 `<ul>` 内部的 `<li>`（宝马、奔驰变绿色），`<ol>` 里的 `<li>`（奥迪）不受影响 |

### 子代选择器

仅选择某元素的**直接子元素**，通过 `>` 连接父级与子级选择器，对孙元素及更深层级的元素无效。

| 项目          | 内容                                                         |
| ------------- | ------------------------------------------------------------ |
| **语法**      | `E > F { ... }` （E 是父级元素，F 是 E 的直接子元素）        |
| **HTML 结构** | `html <div> <a href="#">子元素1</a> <p><a href="#">孙元素</a></p> <a href="#">子元素2</a> </div> ` |
| **CSS 代码**  | `css div > a { color: red; } `                               |
| **效果说明**  | 仅`<div>`的直接子元素`<a>`（子元素 1、子元素 2）变为红色，`<p>`内的`<a>`（孙元素）不受样式影响 |

### 相邻兄弟选择器

用于选择**紧跟在某元素之后的第一个同级兄弟元素**，通过 `+` 连接两个选择器，仅匹配相邻的第一个兄弟元素。*只能往下生效*

| 项目          | 内容                                                         |
| ------------- | ------------------------------------------------------------ |
| **语法**      | `E + F { ... }` （E 是参考元素，F 是紧跟 E 的第一个同级兄弟元素） |
| **HTML 结构** | `html <h1>h1元素</h1> <p>第一个元素</p> <p>第二个元素</p> `  |
| **CSS 代码**  | `css h1 + p { color: red; } `                                |
| **效果说明**  | 仅`<h1>`后**第一个**`<p>`（第一个元素）变为红色，第二个`<p>`不受样式影响 |

### 通用兄弟选择器

用于选择**某元素之后的所有同级兄弟元素**，通过 `~` 连接两个选择器，匹配所有后续的同级兄弟元素（不限个数）

| 项目          | 内容                                                         |
| ------------- | ------------------------------------------------------------ |
| **语法**      | `E ~ F { ... }` （E 是参考元素，F 是 E 之后的所有同级兄弟元素） |
| **HTML 结构** | `html <h1>h1元素</h1> <p>第一个元素</p> <p>第二个元素</p> `  |
| **CSS 代码**  | `css h1 ~ p { color: red; } `                                |
| **效果说明**  | `<h1>`后**所有**`<p>`（第一个元素、第二个元素）都变为红色，区别于相邻兄弟选择器仅匹配第一个 |

</div>

## 弹性盒子

大盒子里的子元素如何摆放

| 概念                           | 说明                                                         |
| ------------------------------ | ------------------------------------------------------------ |
| **弹性容器（Flex container）** | 通过给元素设置`display: flex`定义，是弹性子元素的父容器，用于承载和控制子元素布局 |
| **弹性子元素（Flex item）**    | 弹性容器内的直接子元素，会遵循弹性容器的布局规则排列，容器外和子元素内部的内容正常渲染#### |

### 父元素上的属性

#### display属性

| 取值          | 作用                                                         |
| ------------- | ------------------------------------------------------------ |
| `flex`        | 开启块级弹性盒布局，父元素变为弹性容器，子元素默认**水平排列** |
| `inline-flex` | 开启行内弹性盒布局，弹性容器仅占内容宽度，子元素仍遵循弹性布局规则 |

#### flex-direction属性

| 取值             | 定义               | 排列效果                                     |
| ---------------- | ------------------ | -------------------------------------------- |
| `row`            | 横向排列（默认值） | 从左到右水平排列，左对齐                     |
| `row-reverse`    | 反转横向排列       | 从右到左水平排列，右对齐，最后一项排在最前面 |
| `column`         | 纵向排列           | 从上到下垂直排列                             |
| `column-reverse` | 反转纵向排列       | 从下到上垂直排列，最后一项排在最上面         |

#### justify-content属性

| 取值                 | 核心效果                       | 补充说明                                                     |
| -------------------- | ------------------------------ | ------------------------------------------------------------ |
| `flex-start`（默认） | 弹性项向主轴**起始端**紧挨填充 | 第一个弹性项的外边距线与容器主轴起始边线重合，后续项依次排列 |
| `flex-end`           | 弹性项向主轴**结束端**紧挨填充 | 第一个弹性项的外边距线与容器主轴结束边线重合，后续项反向依次排列 |
| `center`             | 弹性项在主轴**居中**紧挨填充   | 剩余空间均分在弹性项左右 / 上下；若空间不足，弹性项向两侧溢出 |

#### align-items属性

| 取值         | 核心效果                   | 补充说明                                                     |
| ------------ | -------------------------- | ------------------------------------------------------------ |
| `flex-start` | 弹性项向侧轴**起始端**对齐 | 弹性项的侧轴起始边界与容器侧轴起始边界重合                   |
| `flex-end`   | 弹性项向侧轴**结束端**对齐 | 弹性项的侧轴起始边界与容器侧轴结束边界重合                   |
| `center`     | 弹性项在侧轴**居中**对齐   | 剩余空间均分在弹性项上下 / 左右；若容器侧轴尺寸不足，弹性项向两侧等距溢出 |

### 子元素上的属性

#### flex

设置flex属性后宽度不再生效

| 规则       | 说明                                                         |
| ---------- | ------------------------------------------------------------ |
| 基础规则   | `flex: 0`（默认）：即使容器有剩余空间，子元素也不放大        |
| 单元素分配 | 仅一个子元素设置`flex时，该元素会按设置的比例占据**所有剩余空间**（如`flex: 1`占 100%，`flex: 0.5`占 50%） |

## 文档流

### 浮动

float:left;

float:right;

| 属性值    | 描述                         |
| --------- | ---------------------------- |
| **left**  | 元素向左浮动，脱离标准文档流 |
| **right** | 元素向右浮动，脱离标准文档流 |

#### 浮动的核心原理（关键点）

1. **脱离文档流**：这是浮动最核心的特征。浮动元素不再占据标准文档流中的空间，如同 “漂浮” 在上方，导致下方的元素填补上来。
2. **单向性**：浮动**只有左右方向**，没有上下浮动（无法实现垂直居中或向上漂浮）。

#### 视觉效果解析

- **未脱离文档流**：元素按正常顺序排列，占据页面空间，互不重叠。
- **脱离文档流**：元素漂浮于上方，底层元素会向上顶起，从而产生**折叠 / 重叠**现象。

### 清除浮动

| 问题现象           | 具体原因                                                     | 视觉表现                               |
| ------------------ | ------------------------------------------------------------ | -------------------------------------- |
| **父元素高度塌陷** | 浮动元素脱离了标准文档流，不再撑起父容器的高度               | 父容器背景色、边框无法显示，高度变为 0 |
| **后续元素受影响** | 浮动元素漂浮于上层，后续的标准流元素会向上攀爬，占据其原有位置 | 布局混乱、文字 / 图片环绕或重叠        |

#### 解决方案

| 方案名称                               | 实现原理                                                 | 代码示例                                                     | 适用场景                             |
| -------------------------------------- | -------------------------------------------------------- | ------------------------------------------------------------ | ------------------------------------ |
| **父元素设置高度**                     | 手动给父容器指定高度，强制撑起塌陷的父元素               | `.container { height: 100px; }`                              | 子元素高度固定的静态布局（如导航栏） |
| **受影响元素加 clear 属性**            | 给浮动元素的后续元素设置`clear`，阻断浮动影响            | `.content { clear: both; }`                                  | 简单布局，后续元素无需跟随浮动       |
| **overflow 清除浮动（与clear一起用）** | 给父元素设`overflow: hidden/auto`，触发 BFC 包裹浮动元素 | `.container { overflow: hidden;clear：both； }`              | 大多数常规布局，快捷高效             |
| **伪对象方式（伪元素）**               | 用`::after`伪元素在父元素末尾插入内容并清除浮动          | `.clearfix::after { content: ""; display: block; clear: both; }` | 工程化项目，无冗余标签，通用性最强   |

## 定位

position属性指定了元素的定位类型

| 属性值       | 定位类型 | 是否脱离文档流 | 参考定位点                                                   | 核心特征                                   |
| ------------ | -------- | -------------- | ------------------------------------------------------------ | ------------------------------------------ |
| **relative** | 相对定位 | ❌ 不脱离       | 元素自身原本的位置                                           | 偏移后仍保留原文档流空间，不会影响其他元素 |
| **absolute** | 绝对定位 | ✅ 完全脱离     | 最近的**已定位祖先元素**（如`relative/absolute/fixed`），若无则为浏览器视口 | 不占文档流空间，会漂浮于上层               |
| **fixed**    | 固定定位 | ✅ 完全脱离     | 浏览器视口（viewport）                                       | 滚动页面时位置保持不变，常用于导航栏、弹窗 |

设置`position`后，需通过**方向属性**调整元素位置，包括：

- `left`：元素左边缘与参考点的水平距离
- `top`：元素上边缘与参考点的垂直距离
- `right`：元素右边缘与参考点的水平距离
- `bottom`：元素下边缘与参考点的垂直距离

**查找逻辑**

相对 / 绝对定位元素的位置参考点，会**优先找最近的、设置了定位（`position` 不为 `static`）的父级元素**；若父级无定位，则向上遍历祖父、曾祖父等元素，直到找到有定位的祖先，若所有祖先均无定位，则参考**顶层文档（`<html>` 根元素）**。

**堆叠顺序**

Z-index

| 特征         | 说明                                                         |
| ------------ | ------------------------------------------------------------ |
| **取值**     | 整数（正、负、0 均可）。例如 `z-index: 10;` 或 `z-index: -1;` |
| **默认值**   | `auto`（表示不创建新的层叠上下文，继承父级的堆叠顺序）       |
| **生效前提** | **元素必须已定位**（`position: relative/absolute/fixed/sticky`）。❌ 静态定位元素（`static`）设置 `z-index` 无效。 |

## 动画

### CSS 动画核心定义

CSS 动画的本质是**让元素从一种样式平滑过渡到另一种样式**，支持修改任意数量的样式属性，且可通过百分比 / 关键词控制变化的时间节点。

| 规则要点     | 具体说明                                                     |
| ------------ | ------------------------------------------------------------ |
| **样式变化** | 可修改颜色、尺寸、位置、透明度等任意 CSS 样式，且能设置多次变化 |
| **时间节点** | 1. 用**百分比**定义：`0%` 为动画开始，`50%` 为中间，`100%` 为结束2. 用**关键词**：`from` 等价于 `0%`，`to` 等价于 `100%` |

### css动画实现步骤

**CSS 动画实现步骤（核心语法）**

实现 CSS 动画需两个核心部分：**`@keyframes` 定义动画序列** + **动画属性调用**

#### 定义动画序列

/* 方式1：用from/to关键词 */*

@keyframes move {  

​	from { transform: translateX(0); } /* 开始状态 */

​	 *to { transform: translateX(200px); } /* 结束状态 */ 

} *

*/* 方式2：用百分比定义多阶段 */ 

*@keyframes colorChange { *

​	0% { background: red; }  

​	50% { background: yellow; transform: scale(1.5); } /* 中间状态 */ 

​	 100% { background: blue; transform: scale(1); }

 }

#### animation执行动画

animation: name duration timing-function delay iteration-count direction play-state fill-mode;

| 参数名                | 核心作用                                                     | 常用取值                                                     |
| --------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **name**              | 绑定 @keyframes 定义的动画名称                               | 自定义名称（如 `move` `fade`）                               |
| **duration**          | 动画单次持续时间                                             | `1s`/`2000ms`（必须设置，否则无动画）                        |
| **timing-function**   | 动画速度曲线                                                 | `ease`（默认，慢 - 快 - 慢）、`linear`（匀速）、`ease-in`（加速）、`ease-out`（减速）、`ease-in-out`（先加速后减速）、`cubic-bezier(n,n,n,n)`（自定义曲线） |
| **delay**             | 动画延迟执行时间                                             | `0s`（默认）、`2s`（延迟 2 秒执行）                          |
| **iteration-count**   | 动画循环次数                                                 | `1`（默认）、`3`、`infinite`（无限循环）                     |
| **direction**         | 动画播放方向                                                 | `normal`（默认，正向）、`reverse`（反向）、`alternate`（往返，奇数次正向 / 偶数次反向）、`alternate-reverse`（反向往返） |
| **play-state**        | 控制动画播放状态div：hover{      animation-play-state：paused} | `running`（默认，播放）、`paused`（暂停）                    |
| **fill-mode**（补充） | 动画结束后元素的样式状态                                     | `none`（默认，回到初始）、`forwards`（保持结束样式）、`backwards`（提前应用初始样式）、`both`（结合 forwards+backwards） |

##### timing-function值

| 值              | 描述             | 形象理解                                         |
| --------------- | ---------------- | ------------------------------------------------ |
| **ease**        | 逐渐变慢（默认） | 先快后慢，符合自然物体运动惯性                   |
| **linear**      | 匀速             | 全程速度一致，无快慢变化                         |
| **ease-in**     | 加速             | 从慢到快，适用于进入动画                         |
| **ease-out**    | 减速             | 从快到慢，适用于退出动画                         |
| **ease-in-out** | 先加速后减速     | 两头慢，中间快，兼顾进入与结束的平滑（呼吸效果） |
